# Definition of Done (DoD)

Project: Website Quản lý tổ chức sự kiện nội bộ

Definition of Done là tiêu chí để xác định một task hoặc user story đã hoàn thành.

---

# 1. Code Quality

- Code phải được review bởi ít nhất 1 thành viên trong nhóm
- Không có lỗi nghiêm trọng
- Tuân thủ coding standards (ESLint)

---

# 2. Functionality

- Chức năng hoạt động đúng theo yêu cầu
- Không gây lỗi cho các chức năng khác
- API trả dữ liệu đúng format

---

# 3. Testing

- Unit test được viết cho các chức năng quan trọng
- Tất cả test phải pass

---

# 4. Documentation

- Code có comment cần thiết
- README được cập nhật nếu có thay đổi

---

# 5. Git Workflow

- Code được push lên feature branch
- Tạo Pull Request
- Sau khi review mới merge vào develop

---

# 6. Deployment Readiness

- Không có lỗi build
- Có thể chạy được trên môi trường development

---

# 7. Demo

- Chức năng có thể demo cho Product Owner hoặc giảng viên
